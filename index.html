Introduction

HTML, short for HyperText Markup Language, is a vital code that forms the foundation of web pages and their content structure. It enables you to organize and arrange various elements, including paragraphs, bullet points, images, and data tables. By utilizing HTML, you can effectively structure and present information on the web.

In this documentation page, we will guide you through the basics of HTML, providing you with a clear understanding of its core principles and functionality. By the end of this article, you'll have a solid foundation in HTML and be equipped to create well-structured web pages. So, let's delve into the world of HTML and explore its essential features.

What is HTML?

HTML, which stands for HyperText Markup Language, is a markup language used to define the structure of web page content. It consists of a series of elements that enclose different parts of the content and specify their presentation and behavior. By using enclosing tags, you can transform text, images, and other elements on your page. Let's explore an example to illustrate this:

Consider the following line of content:

I love coding!

To indicate that this line is a paragraph, we can enclose it with paragraph tags:

HTML:

<p>I love coding!</p>

Anatomy of an HTML Element

Let's examine the components of an HTML element using the example of a paragraph element:

HTML:

<p>I love coding!</p>

The essential parts of this element include:

  1. Opening Tag: It marks the beginning of the element and is enclosed in angle brackets (< and >). In this case, the opening tag for a paragraph element is <p>.

  2. Closing Tag: It denotes the end of the element and is similar to the opening tag, but with a forward slash (/) before the element name. For the paragraph element, the closing tag is <p>.

  3. Content: It represents the actual text or other elements contained within the opening and closing tags. In our example, the content is "I love coding!".

  4. Elements: The combination of the opening tag, content, and closing tag forms the complete element, which, in this case, is a paragraph element.

Elements can also have attributes, providing additional information about the element. Attributes are defined within the opening tag and consist of a name-value pair. Let's see an example:

HTML:

<a href="https://www.example.com">Visit Example Website</a>

In this case, the `<a>` element represents a hyperlink, and the `href` attribute specifies the URL it links to.

Nesting Elements

HTML allows elements to be nested inside one another, creating a hierarchical structure. Let's consider an example where we want to emphasize a word within a paragraph:

HTML:

<p>I <strong>strongly<strong> recommend learning HTML.</p>

In the above example, the `<strong>` element is nested within the paragraph element (`<p>`), highlighting the word "strongly."

Proper nesting is crucial to ensure elements are correctly opened and closed in the right order.

Void Elements

Some elements in HTML are void elements, meaning they don't have a closing tag and don't contain any content. An example is the
element used to insert line breaks:

HTML:

<p>This is the first line.<br>This is the second line.</p>

In the above code, the `<br>` element inserts a line break within the paragraph.

Anatomy of an HTML Document

Let's examine the structure of a basic HTML document:

HTML:

<!DOCTYPE html> <html lang="en"> <head> <title>My Web Page</title> <meta charset="utf-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <link rel="stylesheet" href="styles.css"> <script src="script.js"></script> </head> <body> <h1>Welcome to My Web Page</h1> <p>This is a sample paragraph.</p> </body> </html>

The HTML document consists of several elements:

  • <!DOCTYPE html>: This declaration specifies the HTML version being used.
  • <html></html>: The root element that encloses the entire HTML content.
  • <head></head>: This element contains meta-information, stylesheets, scripts, and the document title.
  • <title></title>: It sets the title of the web page.
  • <meta charset="utf-8">: This element specifies the character encoding for the document.
  • <meta name="viewport" content="width=device-width, initial-scale=1.0">: The viewport element in HTML ensures that web pages are displayed correctly on mobile devices. It sets the page width to match the viewport width, preventing mobile browsers from rendering pages wider than the viewport and then scaling them down.
  • <link rel="stylesheet" href="styles.css">: It links an external CSS file to style the HTML content.
  • <script src="script.js"></script>: This element includes an external JavaScript file.
  • <body></body>: The body element contains the visible content of the web page.

By combining these elements, you can create well-structured and interactive web pages using HTML.

Images

Let us take a look at the <img> element.

HTML:

<img src="images/my-test-image.png" alt="test" />

The `<img>` element in HTML is used to embed an image into a web page. It uses the src attribute to specify the path to the image file. Additionally, the alt attribute is included to provide alternative text for users who cannot see the image.

The alternative text, or alt text, serves two main purposes.

  1. First, it is read aloud by screen readers for visually impaired users.
  2. Second, it is displayed when the image fails to load. It is important to provide descriptive alt text that conveys the meaning or content of the image.

In the example provided, the alt text "test" is not sufficient. A more appropriate alt text for the image could be "White box with the text: 'My test image'".

This alt text provides a more descriptive and accurate representation of the image. It clearly mentions that the image is a white box with the specific text "My test image" displayed on it. By including the quotation marks around the text, it conveys that the text is part of the image itself.

Using this alt attribute will ensure that users who cannot see the image, whether due to visual impairments or other reasons, can understand the content and context of the image.

Mark up text

In this section, we will delve into important HTML elements that are fundamental for marking up text.

Headings

Heading elements allow you to designate certain parts of your content as headings or subheadings, mirroring the structure of titles, chapters, and subtitles in a book. HTML provides six heading levels, `<h1>` to `<h6>`, although in practice, you typically use only three to four levels at most.

HTML heading elements (`<h1>` to `<h6>`) provide a hierarchy of headings, ranging from the highest level (`<h1>`) for main titles to the lowest level (`<h6>`) for minor subheadings, allowing for structured organization of content.

Additionally, the font size of HTML heading elements decreases as the heading level increases, with `<h1>` being the largest and `<h6>` being the smallest. Each page should contain no more than one `<h1>` elements, but subheadings (`<h2>` through `<h6>`) may be used multiple times per page.

HTML:

4 heading levels: <h1> Main title </h1> <h2> Top level heading </h2> <h3> Subheading </h3> <h4> Sub-subheading </h4>

Paragraphs

The `<p>` element in HTML is used to create paragraphs of text on a webpage. It acts as a container that wraps around a block of text, allowing us to structure and separate our content into meaningful paragraphs. By enclosing our sentences or paragraphs within `<p>` tags, we can control the spacing and formatting of the text, making it easier to read and understand.

HTML:

<p> This is a single paragraph </p>

Lists

Lists are a common way to present information on the web, and HTML provides special elements to structure lists. When working with lists, we need at least two elements. The two main list types are unordered and ordered lists.

Unordered lists are used when the order of the items doesn't matter, such as a shopping list. These lists have bullet points to mark each item. To create an unordered list, we wrap the list items in a `<ul>` element.

Ordered lists are used when the order of the items does matter, such as a set of instructions. These lists are numbered to indicate the order of the items. To create an ordered list, we wrap the list items in an `<ol>` element.

Each item in the list is marked with an `<li>` (list item) element, which represents a single item in the list. With unordered lists, bullet points are automatically added, while ordered lists display numbers for each item, indicating their sequence.

For example, if we wanted to convert the following paragraph fragment into a list:

HTML:

<p> The United States Federal Government is comprised of the Executive branch, Judicial branch, and the Legislative branch...</p>

We could modify the mark up to this:

HTML:

<p> The United States Federal Government is comprised of the </p> <ul> <li> Executive branch </li> <li> Judicial branch </li> <li> Legislative branch </li> </ul>

Which would result in an unordered list which looked like this:

The United States Federal Government is comprised of the

  • Executive branch
  • Judicial branch
  • Legislative branch

If we wanted a numbered list, we would use an ordered list like this:

HTML:

<p> The United States Federal Government is comprised of the </p> <ol> <li> Executive branch </li> <li> Judicial branch </li> <li> Legislative branch </li> </ol>

Which would give us an ordered list like this:

The United States Federal Government is comprised of the

  1. Executive branch
  2. Judicial branch
  3. Legislative branch
Links

HTML links, also known as hyperlinks, are elements that allow users to navigate between web pages. When a user clicks on a link, it triggers an action that directs the browser to load the linked page.

Links are created using the <a> element, where the href attribute specifies the URL (Uniform Resource Locator) of the destination page.

The href attribute stands for "Hypertext Reference" and provides the address or location of the linked resource.

To create a link, you enclose the link text or other elements within the <a> tags and set the href attribute to the desired URL.

For example:

HTML:

<a href="https://example.com">Click here</a>

creates a link that displays "Click here" and leads to the "https://example.com" webpage when clicked.

Conclusion

In conclusion, mastering the basics of HTML is crucial for web development. By understanding elements such as headings, paragraphs, lists, and links, you gain the ability to create well-structured and engaging web pages. HTML forms the foundation for designing and presenting content effectively on the web.

Headings allow you to establish a hierarchy and emphasize important sections, while paragraphs provide a means to convey clear and concise textual information.

Lists offer a way to present information in an organized and visually appealing manner, whether through bullet points or numbered format.

Moreover, links play a vital role in facilitating seamless navigation between web pages, enhancing the user experience.

With a strong grasp of HTML, you unlock endless possibilities for designing captivating and interactive websites.
